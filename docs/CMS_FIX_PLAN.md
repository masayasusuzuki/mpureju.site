# Instagram URL ズレの原因・影響・修正方針

## 1. 原因

症例記事のバッチ処理スクリプト（`cms-bulk-post.sh` + `microcms-case-post.js`）が `public/case/urls.txt` を上から順に処理し、1行ごとに1ディレクトリ（001, 002, 003...）を作成していた。

このバッチ処理中に **urls.txt の #51〜#54 の4件が2回処理された**。

- 1回目: dir 051〜054 が作成された（正常）
- 2回目: dir 055〜058 が **同じURL** で再度作成された（重複）

結果、dir 055 以降のすべてのディレクトリで「ディレクトリ番号」と「urls.txt の行番号」がズレた。本来 dir 055 に入るべき urls.txt #55 のURLは dir 059 に入り、以降すべて4つずつ後ろにずれた。

さらに、処理中に以下の空ディレクトリも生成されたため、ズレは段階的に拡大した：

| 事象 | ズレの増分 | 累計ズレ |
|------|-----------|---------|
| urls.txt #51〜54 が2回処理（dir 051-054 と dir 055-058 が重複） | +4 | **+4** |
| dir 091 が空ディレクトリとして生成（URLを消費せずカウンターだけ進んだ） | +1 | **+5** |
| dir 135 が同様に空ディレクトリとして生成 | +1 | **+6** |

## 2. ズレの影響範囲

### 2-1. ローカルファイル（article.md + CASE_INDEX.md）

| ディレクトリ範囲 | ズレ量 | 状態 |
|---|---|---|
| dir 001〜054 | 0 | 正常 |
| dir 055〜090 | +4 | article.md の instagram_url が4つ分古いURLになっている |
| dir 092〜134 | +5 | 同上、5つ分 |
| dir 136〜150 | +6 | 同上、6つ分 |

**ズレているのは instagram_url のみ**。記事のタイトル・本文・画像・サムネイルは正しい。

→ これは**修正済み**（2026-04-02 に article.md および CASE_INDEX.md を一括修正）

### 2-2. microCMS（cases API）

article.md の内容をそのまま microCMS に POST したため、CMS 側にも同じ instagram_url のズレがそのまま入っている。

加えて、以下の追加問題がある：

**重複登録 — 同じ slug の記事が CMS に2件存在する（計4 slug、8件）**

dirs 051-054 の重複処理で、本来とは異なる内容が先に POST された。その後 dirs 055+ から正しい内容が POST されたが、同じ slug で2件目として登録された。

| slug | 正しい記事の contentId | 削除すべき重複の contentId | 重複側のタイトル（誤り） |
|------|----------------------|--------------------------|----------------------|
| skin-011 | huf1li38mr | sjn70366b | スキンブースターの肌育で毛穴の赤みと乾燥・炎症を改善した症例 |
| eye-017 | a2st8y6h3 | w2cul_pzbw6 | リガメント処理を含む裏ハムラで脱脂後に改善しなかった目の下のへこみを解消した症例 |
| eye-018 | y-gt9v-cx | 1wtigqo6nff2 | 二重全切開修正のダウンタイム経過（術直後〜1ヶ月）を解説した症例 |
| skin-012 | 6_vgzsmfby8 | 6m5hjl0_l | サーマジェン（RF）で顔の引き締めと小顔効果を実現した症例（施術直後） |

**追加の重複 — 別の原因で同じ slug が2件登録されている（計3 slug、6件）**

| slug | contentId（2件） |
|------|-----------------|
| mouth-026 | a-1pao6tyoq, ktr12dspc0gg |

| skin-016 | h0jk92h4f, jgyugx-zegsy |

これらは dirs 051-054 とは無関係の重複。投稿スクリプトの二重実行等が原因と推測。

## 3. 修正方針

### ステップ1: ローカルファイル修正 ✅ 完了

dir 055〜150 の article.md 内の `instagram_url` を、urls.txt のディレクトリ番号に対応する正しいURLに差し替え済み。CASE_INDEX.md も同期済み。

### ステップ2: microCMS 重複記事の削除（DELETE）

以下の7件を microCMS Management API で削除する。

| No | slug | 削除する contentId | 理由 |
|----|------|--------------------|------|
| 1 | skin-011 | sjn70366b | dir 051 由来の重複 |
| 2 | eye-017 | w2cul_pzbw6 | dir 052 由来の重複 |
| 3 | eye-018 | 1wtigqo6nff2 | dir 053 由来の重複 |
| 4 | skin-012 | 6m5hjl0_l | dir 054 由来の重複 |
| 5 | mouth-026 | 要確認（どちらが正か） | 二重投稿による重複 |
| 6 | mouth-034 | 要確認（どちらが正か） | 二重投稿による重複 |
| 7 | skin-016 | 要確認（どちらが正か） | 二重投稿による重複 |

※ mouth-026, mouth-034, skin-016 は CMS 上で内容を確認し、正しい方を残す。

### ステップ3: microCMS の instagram_url 修正（PATCH）

残った記事（89件）の `instagram_url` フィールドを、修正済みの article.md の値で上書きする。

対象は dir 055〜150 に対応する全 cases 記事。タイトル・本文・画像はすべて正しいため、`instagram_url` フィールドのみを PATCH する。

### ステップ4: 検証

修正後、microCMS の全 cases を再取得し、ローカルの article.md と照合。slug・instagram_url が全件一致することを確認する。

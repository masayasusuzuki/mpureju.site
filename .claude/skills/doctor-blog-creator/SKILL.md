# Doctor Blog Creator

Instagramの院長投稿（画像＋キャプション）から、microCMS blogスキーマに準拠した院長ブログ記事（article.md）を生成するスキル。

---

## コンセプト

院長ブログは**専門医としての知見と人柄**を伝えるコンテンツ。
スタッフブログよりも専門的・教育的な内容だが、堅すぎず患者目線で書く。
Instagram投稿の内容を深掘り・補足し、Web記事として価値ある情報に仕上げる。

---

## ワークフロー

### Step 1: 対象ディレクトリの特定

ユーザーが指定したディレクトリ（例: `public/doctor/001/`）を確認する。
指定がない場合は `public/doctor/` 配下で `article.md` がまだないディレクトリを探す。

### Step 2: 画像とキャプションを読み込む

**画像：** ディレクトリ内の画像ファイル（.jpg, .jpeg, .png）をすべて番号順に読み込む。
各画像から以下を抽出する：
- テキスト・見出し・箇条書きの内容
- 施術名・解剖学的な情報・医療用語
- Before/After・症例写真・図解

**キャプション（caption.txt）：** Instagramの投稿文。
院長の語り口を読み取り、記事のトーンに反映する。

### Step 3: frontmatterを設定する

microCMS blogスキーマに準拠したYAML frontmatterを生成する。

```yaml
---
title: "記事タイトル（専門性と親しみやすさを両立、30〜50文字程度）"
slug: "doctor-{連番3桁}"
category:
  - "カテゴリ名"
tags: "タグ1,タグ2"
thumbnail: "最初の画像ファイル名"
instagram_url: ""
published_at: "YYYY-MM-DD"
---
```

#### スラグ命名規則

`doctor-{連番3桁}`（例: `doctor-001`, `doctor-002`）。
連番はディレクトリ番号と一致させる。

#### カテゴリ一覧（複数選択可）

- `症例解説` — Before/After解説、施術の工夫
- `美容医療知識` — 解剖学、エイジング、施術原理
- `学会・研修` — 学会参加レポート、最新知見
- `日常` — 院長のプライベート、趣味

#### タグ

記事内容に応じたキーワードをカンマ区切りで設定。
施術名、部位名、成分名などSEOに寄与する具体的なワードを選ぶ。

### Step 4: 記事本文を生成する

#### 構成ルール

1. **導入（1〜2段落）** — 読者の関心を引く問いかけ or テーマ提示
2. **本論（h2で2〜4セクション）** — 専門知識をわかりやすく解説
3. **まとめ（1段落）** — 要点の整理 + 相談への誘導

#### 文体ルール

- 一人称は「私」、読者は「患者様」「皆様」
- 専門用語を使う場合は必ず平易な言い換えを併記
- 断定は避ける（「〜が期待できます」「〜と考えています」）
- 医療広告ガイドライン遵守（効果保証しない、リスクに触れる）

#### Markdown記法

- 見出しは `##`（h2）のみ使用。h3以下は使わない
- **本文中に画像参照（`![alt](ファイル名)`）を書かない** — 画像はサムネイルとimagesフィールド（カルーセル）で表示する。本文はテキストのみ
- 強調は `**太字**` を控えめに使用

### Step 5: article.md を書き出す

frontmatter + 本文をMarkdown形式で `article.md` に書き出す。

### Step 6: ディレクトリをリネーム

記事タイトルの要約（15文字以内）でリネームする。

```
public/doctor/001  →  public/doctor/001_ヒアルロン酸の選び方
```

### Step 7: インデックスを更新する

以下の2ファイルに記事情報を追記する：
- `public/doctor/DOCTOR_INDEX.md` — テーブルに行を追加
- `public/ARTICLE_INDEX.md` — 院長ブログセクションのテーブルに行を追加

### Step 9: 投稿日を設定する

`instagram_url` に元投稿のURLを設定する。
投稿日（`published_at`）はInstagramの投稿日に合わせる。

#### 投稿日の取得方法

`published_at.txt` がディレクトリ内にあればその内容（`YYYY-MM-DD`）を使う。
なければ、instaloaderのダウンロードファイル名（例: `2024-12-14_03-41-31_UTC_1.jpg`）の先頭から `YYYY-MM-DD` を抽出する。
どちらもなければユーザーに確認する。

## microCMS投稿

記事完成後、以下で投稿する：

```bash
export $(grep -v '^#' .env.local | grep -v '^image' | xargs) && node scripts/microcms-doctor-blog-post.js "public/doctor/{ディレクトリ}"
```

投稿スクリプトは以下を自動処理する：
- `thumbnail`: frontmatterで指定した画像をアップロード
- `images`: サムネ以外の画像（image_2.jpg等）をすべてアップロードして `images` フィールドに設定
- 本文中の画像参照（`![alt](image_2.jpg)`）をmicroCMS画像URLに差し替え

---

## 注意事項

- **医療広告ガイドライン必須** — 院長ブログは医療情報を含むため、断定表現・効果保証は絶対に使わない
- 本文はMarkdownのまま保存（HTML変換しない）
- 画像参照のファイル名は実在するファイルと一致させる
